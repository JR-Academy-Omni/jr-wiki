01 · OpenAI 旗舰发布
GPT-5.6 三版本齐发，Sol旗舰代码效率提升54%
图 2 · 海报 01
一句话：OpenAI 发布 GPT-5.6 系列，代码任务 token 效率提升 54%

OpenAI 本周将 GPT-5.6 全量放开，终结了此前按美国政府要求分批灰度的限制。此次发布没有单一旗舰，而是三档并行——Sol 专为深度任务打造，内置「ultra」模式允许模型将复杂请求拆分给各类专用子模型协同完成；Luna 追求极致速度，适合对时延敏感的实时对话场景；Terra 在性能与效率之间取得平衡，作为大多数日常工作的首选。

Sol 的代码 token 效率指标提升 54%，这一数字来自 OpenAI 官方公布的 agentic coding 基准对比，意味着同等代码生成质量下调用成本降低了约一半。这对企业级 CI/CD 流程和大规模代码审查场景具有直接的降本价值，也是本次发布中最具商业说服力的数据点。

与模型同步上线的还有 ChatGPT Work 工具，整合团队工具上下文、支持跨协作者共享，可在桌面端、web 端及移动端（Plus/Pro/Business/Enterprise/Edu）使用。此前的 Atlas 浏览器 Agent 将于 8 月 9 日下线，OpenAI 将同类能力迁移到 ChatGPT 主客户端和 Codex 之中，统一 Agent 入口。

来源：axios.com · openai.com
02 · DeepSeek 破戒融资
DeepSeek首轮外部融资510亿元，同步曝光自研AI芯片计划
图 3 · 海报 02
一句话：创中国 AI 单轮融资纪录，同时接触芯片设计与晶圆厂伙伴

DeepSeek 自成立以来一直以「低调务实」著称，创始人梁文锋多次在公开场合表示不会做外部融资、不谋求上市。此次完成 510 亿元人民币（约 70 亿美元）的首轮外部融资，估值逼近 4000 亿元，打破了这一铁律，在中国 AI 圈引发广泛讨论——这不仅是单轮融资的金额纪录，更是信号意义的转变：从「独立学术派」向「重基础设施扩张」的方向移动。

资金的去向与同期曝光的自研芯片计划密切相关。据路透社援引知情人士消息，DeepSeek 正在研发专用于 AI 推理的定制芯片，当前仍处于早期阶段，正与芯片设计公司、晶圆代工厂和存储厂商展开技术合作洽谈。推理芯片（而非训练芯片）是其主攻方向，因为 DeepSeek 的核心优势一直在于极致的推理效率——V3 和 R1 系列都以极低算力成本取得了令人瞩目的性能表现。

如果自研芯片路线走通，DeepSeek 将进一步摆脱对英伟达 GPU 的依赖，在中美半导体管制博弈加剧的背景下意义深远。与此同时，V4 正式版预计于本月中旬上线，将引入峰谷定价机制——高峰期与非高峰期 API 价格分档，对高频调用的开发者而言值得提前规划请求时间安排。

来源：bloomberg.com · pandaily.com · usnews.com
03 · xAI 代码旗舰
Grok 4.5发布：收购Cursor后第一款代码专属旗舰
图 4 · 海报 03
一句话：主打工程与知识工作，输入 $2 / 百万 token，输出 $6

Grok 4.5 是 xAI 完成对 Cursor 收购之后交出的第一份「协同训练」答卷。xAI 将这款模型定位为首款专为编码和 Agent 工作设计的旗舰级产品，在工程类和知识工作基准测试上宣称超越可比模型。定价策略相对激进：输入端 $2 / 百万 token，输出端 $6 / 百万 token，相较于 GPT-5.6 Sol 的旗舰定价存在明显价格空间，为成本敏感的开发者提供了另一选择。

联合训练模式是本次发布最值得关注的技术叙事。Cursor 拥有超过 900 万活跃开发者用户及大量真实代码编辑数据，这些数据被认为是训练代码感知模型的高质量语料。xAI 将 Cursor 的使用数据与模型训练结合，理论上可以让 Grok 4.5 更贴近开发者实际工作流中的编辑、补全、重构等场景，而非仅依赖合成代码数据。

产品端同步更新了 Grok Build——面向开发者的 CLI 和 Agent 构建工具，新增 /goal 命令、刷新了 /sessions 流程、改进了 bash 和文件输出的可靠性，以及在大型代码库上更快的启动速度。语音端则新增 21 款多语言旗舰声音，支持实时 Voice Agent API。

来源：axios.com · felloai.com
04 · Google 旗舰延期
Gemini 3.5 Pro推迟至7月17日，放弃旧架构全面重建
图 5 · 海报 04
一句话：对标 GPT-5.6 与 Fable 5，主打「性价比旗舰」定位

Gemini 3.5 Pro 的延期并非简单的测试延误，而是一次更彻底的战略调整。Google DeepMind 决定放弃现有的 Gemini 2.5 Pro 架构基础，进行全面的「ground-up redesign」，专门针对数学推理、SVG 场景生成和图像质量三个维度做深度改进。这一决定对应的竞争压力显而易见：当前 GPT-5.6 和 Anthropic Fable 5 已占据顶级基准的主要位置，Gemini 模型在多个排行榜上不再稳居前列。

新版本的核心技术特性包括：2 百万 token 的上下文窗口（目前同级产品中最大之一）、Deep Think 推理层（用于复杂多步问题解决）、以及自主工作流执行能力。定价策略上，Google 将 Gemini 3.5 Pro 定位为「成本效益型旗舰」，主要对标企业侧对大上下文、合理价格的需求，而非与 Sol 正面竞争最强性能。

人才流失风险同样值得关注。本周有报道指出，Google DeepMind 两位核心研究员在 48 小时内相继出走，外界担忧连续人才流失可能影响 DeepMind 在前沿 AI 研究上的长期竞争力。架构重建能否在 17 日如期发布、发布后表现能否达到预期，将是近期 Google AI 团队最重要的信号之一。

来源：finance.biggo.com · thursdai.news · fortune.com
05 · Anthropic 全量更新
Claude Sonnet 5成全用户默认模型，Cowork扩展至移动与网页端
图 6 · 海报 05
一句话：性能接近 Opus 4.8，8 月 31 日前维持优惠定价

Anthropic 在 7 月初完成了本年度最大规模的用户侧模型切换：Claude Sonnet 5 正式取代 Sonnet 4.6 成为所有 Free 和 Pro 用户的默认模型，全球范围内立即生效。Sonnet 5 被 Anthropic 定义为「史上最强 Sonnet」，在 Agent 工作流上的表现接近旗舰 Opus 4.8，同时优惠期价格（至 8 月 31 日）还低于上一代 Sonnet 4.6——对 API 调用方而言，这是一个不切换反而多花钱的窗口期。

Cowork 的移动与网页端扩展是另一个值得关注的动向。Cowork 是 Anthropic 推出的异步协作工具，支持后台任务执行、计划调度、共享对话和项目管理。此前仅限桌面端，本次扩展至移动和网页端后，用户可以在手机上发起任务、查看进度、审批 Agent 操作，不再被绑定在桌面前。Max 用户优先进入 Beta，使用限额同期翻倍，截止日期为 8 月 5 日。

此外，Anthropic 研究团队发表了一项内部发现：在 Claude 的处理过程中观测到一个「小型内部工作空间」，Claude 用其在不转化为语言输出的情况下暂存和操作想法，与人类有意识接触思维的方式存在相似之处。这一研究在科学界和伦理圈引发了广泛讨论。科学研究方向上，Anthropic 还推出了 Claude Science 工作台，整合研究人员常用工具包，生成可审计的科研制品。

来源：releasebot.io · nbcnews.com · anthropic.com
