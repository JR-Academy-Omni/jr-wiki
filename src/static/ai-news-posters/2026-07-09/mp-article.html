01 · 模型发布
OpenAI GPT-5.6 全球解禁开放
图 2 · 海报 01
一句话：白宫解除封锁，Sol/Terra/Luna 三档模型同日全球上线

OpenAI 于 7 月 8 日宣布，其 GPT-5.6 系列模型正式面向全球开放。此前，受白宫要求，OpenAI 仅向部分合作伙伴提供了有限预览访问，政府现已正式解除这一限制。GPT-5.6 包含三个模型：Sol（旗舰）、Terra（性价比优化）和 Luna（速度最快、成本最低），构成覆盖不同需求层级的完整产品线。

OpenAI 对 Sol 的定位是相比 GPT-5.5 有「阶跃式提升」的前沿智能体模型，尤其在复杂多步骤任务、代码生成和推理能力上有显著跃升。Terra 面向需要平衡能力与成本的主流开发场景，而 Luna 则以速度见长，Cerebras 加速下可达每秒 750 个 token，适合高并发和延迟敏感型应用。这一产品分层设计和竞争对手 Anthropic 的 Opus/Sonnet/Haiku 体系形成直接对标。

对于使用 OpenAI API 的开发者和企业来说，本次全球开放意味着无需等待邀请即可接入最新旗舰能力。考虑到 GPT-5.6 在智能体任务上的提升，有 agentic workflow 需求的团队应优先评估 Sol；而需要控制成本的业务可以考虑 Luna 作为高吞吐推理后端。值得注意的是，同期有报道指出 Anthropic 年化营收已超越 OpenAI，竞争格局进一步白热化，这也是 OpenAI 加速全球推出的重要背景。

02 · 算法突破
谷歌 Gemini 3.5 Pro 推翻基座从头重建
图 3 · 海报 02
一句话：2M token 窗口 + Deep Think 推理层，7.17 为目标上线日

谷歌 DeepMind 做出了一个代价极高的决定：彻底推翻 Gemini 2.5 Pro 基座，启动全新预训练。根据多方报道，这一决定背后是内部对模型在多步骤数学推理和 SVG 场景生成上存在明显性能瓶颈的担忧。考虑到前沿模型的预训练成本动辄数亿美元，从头重建意味着数月开发时间和巨额资源投入的放弃——但谷歌判断，修补现有架构无法达到竞争目标。

新版 Gemini 3.5 Pro 计划引入三项核心能力升级：200 万 token 超长上下文窗口（当前 Gemini 2.5 Pro 已有 100 万，此次翻倍）、Deep Think 深度推理层（针对复杂数学和逻辑问题）、以及自主工作流能力。这一目标配置直指 OpenAI GPT-5.6 Sol 和 Anthropic Fable 5 的核心优势区间。此外，谷歌还有 Nano Banana Pro（图像生成）和 Gemini 4 Flash（速度优先）两条并行产品线同步推进。

7 月 17 日是目前最广泛报道的发布目标日期，但谷歌官方尚未公布正式 GA 时间。对于正在评估 Gemini 作为生产环境基座的开发者，建议在官方发布确认前保持观望，并在发布后重点测试长上下文文档处理、数学推理链和复杂多步骤 workflow 场景——这些是本次重建的核心攻关方向，也是判断「是否真的值得升级」的最直接依据。

来源：techtimes.com · hackernoon.com · geeky-gadgets.com
03 · 产品更新
Claude Cowork 扩展 Web/Mobile，同日登陆 Azure Foundry
图 4 · 海报 03
一句话：M365 写作工具 + 跨设备同步，企业侧 Azure 原生上线

Anthropic 此轮更新覆盖两条产品线。一是 Claude Cowork 的平台扩展：此前 Cowork 主要在桌面端运行，本次正式覆盖 Web 和 iOS/Android，实现远程会话和文件跨设备同步；更重要的是新增了 Microsoft 365 深度集成——用户可以通过 Claude 直接起草和发送邮件、管理日历、创建或更新 OneDrive 和 SharePoint 文件。Max 计划用户将率先进入 Beta，其他订阅层级随后开放。

二是 Claude 在 Azure 的正式发布：Claude Opus 4.8 和 Claude Haiku 4.5 现已在 Microsoft Azure Foundry 正式 GA，面向企业客户提供 Messages API 访问，支持编码、agentic 工作流和复杂推理任务。Azure 原生发布最大的卖点在于合规——企业可使用现有的 Azure 身份验证、计费和治理体系，无需新增合同或合规流程，并可选择 US 数据区以满足数据主权要求。

对于企业 IT 和开发团队来说，这两条更新共同指向一个方向：Claude 从「独立 AI 工具」转向「嵌入式企业工作流组件」。Cowork + M365 的组合意味着知识工作者可以在不切换工具的情况下调用 AI 能力；Azure 发布则打通了 Claude 与企业现有云基础设施的集成路径。如果你的团队已经重度依赖 Microsoft 生态，这是一次值得认真评估的窗口。

来源：anthropic.com · releasebot.io · aiweekly.co
04 · 语音 AI
xAI 上线 Grok Voice Agent Builder Beta
图 5 · 海报 04
一句话：无代码平台打包电话、检索、工具与护栏，Grok 4.1 同步全量放开

xAI 本次发布的 Voice Agent Builder 是一个面向企业的无代码语音 AI 代理构建平台，内置了生产级语音代理所需的完整技术栈：电话网络接入、知识库检索、工具调用、安全护栏（guardrails）和可观测性（observability）模块。开发者无需自行拼装 ASR + TTS + LLM + 对话管理的技术链路，可直接在 Grok Voice 上配置并上线语音代理，大幅降低语音 AI 产品的部署门槛。

在模型侧，Grok 4.1 本次面向 grok.com、X 平台以及 iOS/Android 应用的全体用户开放 Auto 模式。背景信息值得关注：SpaceX 近期完成了对 xAI 的收购，两家公司的资源整合正在加速，xAI 同时宣布将在 2026 年剩余时间内以月为单位持续从头训练并发布「全新基座模型」。Grok 4.5 目前处于私测阶段，若按这一节奏，公开发布可能不会太远。

对于国内出海企业和技术团队来说，Grok Voice Agent Builder 提供了一条快速试水英语市场语音客服和语音 AI 产品的低成本路径，无需自建语音基础设施。而 Grok 4.1 的免费全量开放，也让个人开发者和中小团队可以在生产环境中免费使用目前 xAI 最新的公开旗舰模型，用于代码、推理和创作任务。

来源：x.ai · llm-stats.com · blog.mean.ceo
05 · 科研工具
Claude Science 科学工作台开放，$30K 资助申请截至 7.15
图 6 · 海报 05
一句话：多智能体可复现管道，覆盖基因组/蛋白质/化学信息学

Claude Science 是 Anthropic 面向科学研究场景推出的专用 AI 工作台，目前以 Beta 形式向 Pro、Max、Team 和 Enterprise 用户开放，支持 macOS 和 Linux。与通用 Claude 不同，Claude Science 集成了研究人员最常用的工具包和计算环境，支持基因组学（单细胞 RNA 测序分析）、蛋白质组学（CRISPR 筛选设计、蛋白质结构预测）和化学信息学等方向的全流程工作。关键特性之一是「可审计性」：每一步分析都会产出可追溯的 artifact，满足科研可复现性要求。

早期测试者包括 Manifold Bio、Allen Institute 神经科学家 Jérôme Lecoq，以及 UCSF Brain Tumor Center 的 Stephen Francis，分别在各自的研究方向上验证了 Claude Science 在迭代分析和稿件打磨上的实用价值。Anthropic 同时宣布开放 AI for Science 资助项目，将向最多 50 个入选项目各提供最高 $30,000 的算力积分，申请截止日期为 7 月 15 日，7 月 31 日前通知结果。

对于 AI 方向的留学生、科研人员和高校实验室，这是一个值得认真评估的机会——既可以免费获得可观的算力资源用于毕业论文或课题研究，也是和 Anthropic 建立早期接触的窗口。7 月 15 日截止，时间不多，有研究方向需求的朋友建议尽快准备申请材料。申请入口和项目详情可直接访问 Anthropic 官网的 Claude Science 页面查看。

来源：anthropic.com · techcrunch.com · marktechpost.com
