01 · 价格战
DeepSeek V4-Pro 永久降价75%：API 最低 0.025 元/百万 Token，全球定价格局被迫重写
图 2 · 海报 01
一句话：「临时促销」变「永久底价」——V4-Pro-Max 超越 GPT-5.2 / Gemini 3.0 Pro 部分榜单，现在还更便宜

DeepSeek 此次降价的背景值得细看。今年 4 月推出 V4-Pro 时，公司坦承因「高端算力受限」而不得不对 Pro 版设置最高 12 倍于 Flash 版的溢价，并明确表示一旦华为 Ascend 950 超节点在下半年大规模量产，Pro 价格将大幅下调。现在这个下调提前到来且以「永久」而非「促销」的形式落地，意味着 DeepSeek 对算力成本的控制预期比外界预料的更乐观，也表明中国 AI 的成本优势已从「市场教育期的低价补贴」演变为可持续的结构性竞争力。与此同时，V4-Pro-Max 在多项推理基准测试中声称优于 OpenAI GPT-5.2 和 Gemini 3.0 Pro，「比竞品强还比竞品便宜」的组合正在真正构成威胁。

对于使用 API 构建产品的开发者和企业，这次降价的意义是实质性的。以中等规模的 RAG 系统为例，每月处理 5 亿输入 Token 和 5000 万输出 Token 的成本，采用 V4-Pro 旧价格需要约 3,200 元人民币，新价格仅需约 1,100 元——节省超过 65%。对于澳洲初创企业或正在构建 AI 产品的 JR Academy 学员，「V4-Pro vs GPT-5.5 Instant vs Gemini 3.5 Flash」的成本-性能取舍对比已经值得认真评估，而不是默认选择 OpenAI API。独立基准测试（非 DeepSeek 官方数据）在接下来几周内会给出更客观的评估，届时可结合实际业务场景做决策。

从更宏观的视角看，DeepSeek 的降价行为正在倒逼全球 AI 定价体系重构。Google 在 I/O 2026 上大幅削减 Gemini Ultra 订阅价（从 $250 降至 $200/月）、发布定价极具竞争力的 Gemini 3.5 Flash，背后都有 DeepSeek 价格压力的影子。对于正在学习 AI 产品开发的同学，理解「成本竞争如何重塑 AI 产业格局」将是非常有价值的行业洞察，也是在面试中展示 AI 产业思维深度的好素材。

02 · 安全边界
Claude Mythos 具国家级网攻能力：Anthropic 决定封存不公开，仅授权少数政府和企业修补漏洞
图 3 · 海报 02
一句话：「太强了所以不能给你」——Mythos 是 Anthropic 首个因能力过载而主动封存的模型，网络安全边界被重新定义

Anthropic 的「负责任扩展政策」（RSP）自发布以来一直被外界视为「理论框架」，此次 Claude Mythos 的封存是 RSP 首次在真实模型上触发实质性限制行动。RSP 按照「AI 安全等级」（ASL）对模型危险程度分级：ASL-1 是当前主流模型，ASL-3 是有潜在大规模危害风险、ASL-4 则是能力已超越当前安全措施可管控范围的模型。Mythos 被认定为 ASL-4 的核心原因是其网络进攻能力：在测试中它展示出可被用于国家级网络行动的潜力，包括发现和利用广泛部署软件中的零日漏洞。与其「对齐后公开」，Anthropic 选择了更激进的「不分发」路线。

Anthropic 的应对方案体现了一种「以毒攻毒」的安全思路：既然 Mythos 能攻，就让少数经过严格审核的机构用它来防。这些合作方（政府机构和安全公司）的任务是用 Mythos 主动扫描全球广泛使用的软件基础设施，提前发现并修补漏洞，而不是等攻击者发现漏洞后再响应。这是 AI 在网络安全领域从「辅助分析工具」升级为「主动防御引擎」的一个关键节点。同期 Anthropic 发布的 Claude Security 公测版和 Project Glasswing 网络验证工具，也是这一战略布局的组成部分。

对于学员和从业者，Mythos 的封存有两个值得关注的启示。其一，「AI 能力边界」这个话题从哲学讨论进入了产业现实：如何定义「安全」能力边界、监管机构将如何回应，将成为接下来 12-18 个月 AI 治理领域的核心议题，对政策、合规和企业 AI 采购策略均有直接影响。其二，对于有兴趣进入 AI 安全和红队测试领域的同学，Anthropic ASL 框架和 Claude Constitutional AI 的研究文档是非常好的入门材料，这个方向在澳洲政府和金融机构中需求快速增长。

03 · 产品扩张
OpenAI Codex 登陆手机 + 个人财务 AI 上线：ChatGPT 从「对话框」全面进化为「个人智能基础设施」
图 4 · 海报 03
一句话：两大功能同周亮相——Codex 移动端可随时查看代码任务进度；财务 AI 接入银行账户做资产分析

Codex 移动端的发布代表着「AI 编程 Agent」工作流范式的一次关键跃升。此前，Codex 只能在桌面浏览器中使用，而真实的软件开发场景中，开发者经常需要一边通勤、开会或处理其他事务，一边监控运行中的 AI 编程任务。Codex 移动端解决的正是这个「离开桌面后失联」的问题：用户可以在手机上查看任务执行日志、对关键操作（如文件删除、API 调用）进行审批确认、在 Agent 卡住时提供补充指令，从而将编程任务从「需要坐在电脑前盯着」变为「后台运行、按需介入」。这与 Anthropic Managed Agents 的 webhook 触发逻辑异曲同工——异步 AI 编程工作流正在成为新常态。

个人财务功能的发布则是 ChatGPT 进入「高敏感垂直场景」的重要信号。用户可以安全地连接银行账户、信用卡和投资账户，ChatGPT 基于真实的账户数据（而非用户描述）给出财务建议：「我上个月在哪里超支了？」「我距离存够 6 个月紧急备用金还差多少？」「如果我每月多存 $500，几年后能还清房贷？」这些问题以前需要用户自己整理数据然后输入 ChatGPT，现在数据直接接入后问答效率大幅提升。值得注意的是，OpenAI 选择将财务功能首先向「Pro 用户」而非全体用户开放，暗示对话质量和安全合规门槛在金融场景下依然是优先考量。

对于正在使用或评估 AI 工具的学员，这两个功能传递了一个清晰信号：OpenAI 的战略方向是让 ChatGPT 渗透进用户「真实生活的全部场景」，而不只是回答工作中的问题。Codex 移动端会是在澳洲做软件开发岗位面试时值得展示的实操经验；财务 AI 功能目前仅限美国 Pro 用户，澳洲用户预计在 Q3 2026 逐步获得访问权限，可以提前关注隐私政策和数据主权相关条款。

04 · 编程 Agent
Grok Build 向 SuperGrok 全面开放：Vercel / Canva / S&P 数据一键接入，编程 Agent 生态快速成型
图 5 · 海报 04
一句话：「写代码 + 部署 + 设计 + 数据」全链路打通——SuperGrok 用户无需额外付费即可用 Grok Build 在 OpenCode 环境跑编程任务

Grok Build 和上周的 Grok Skills 是 xAI 在五月集中推进的两条战线：Skills 解决「AI 懂你是谁」，Build 解决「AI 帮你干活」。两者结合，Grok 对开发者的价值主张从「聪明的聊天对象」变成了「可编程的工作助理」。Grok Build 0.1 是专门针对编程任务的微调模型，不同于 Grok 4.3 的通用旗舰定位——它在代码生成、调试、重构和跨文件引用等 Agentic 场景下有针对性优化，通过 OpenCode 环境（xAI 自建的开发者 IDE）运行任务时，可以访问文件系统、执行代码、调用外部 API。SuperGrok 订阅用户无需额外付费即可访问，这是 xAI 对「把最好的工具留给重度用户」这一策略的延续。

第三方连接器是这次发布中更值得关注的部分。Vercel 和 Canva 的接入意味着用户可以在 Grok 对话中直接完成「写代码 → 部署上线」和「生成设计稿」的操作，不需要切出到其他工具。但真正形成差异化的是 S&P Global 实时市场数据的接入——这是 ChatGPT、Claude 和 Gemini 目前均不具备的原生金融数据集成，让 Grok 在量化研究、投资分析和金融科技开发场景中获得独特优势。结合 Grok 4.3 的 100 万 Token 上下文和原生视频输入，xAI 正在逐步构建一个以「实时数据 + 长上下文 + 端到端开发」为核心的产品护城河。

对于正在评估 AI 开发工具的学员，Grok Build 值得动手测试的理由有两个：一是它基于 Grok 4.3 强大的长上下文能力，处理大型代码库时的表现值得与 Claude Code 和 GitHub Copilot 做横向对比；二是如果你的项目涉及金融数据（量化交易、财务分析、投资组合工具），S&P Global 集成是立即可用的稀缺资源。SuperGrok 月费约 $30 美元，对于正在构建 portfolio 项目的求职者，投入产出比值得认真考量。

来源：engadget.com · basenor.com · x.ai
05 · 企业铺开
Claude 进驻 KPMG 全球 27 万员工 + SAP 全线 AI 平台：「Anthropic 企业浪潮」正式开启规模化
图 6 · 海报 05
一句话：不是试点是全铺——KPMG 27 万人人手一个 Claude，SAP 把 Claude 嵌进全球 ERP 核心引擎

KPMG 和 SAP 在同一周宣布与 Anthropic 的深度合作，时间节点并不偶然——这两个公告都在 Code with Claude 2026 开发者大会结束后紧随而来，是 Anthropic 向外界展示「Claude 不只是面向开发者的 API，而是企业级智能基础设施」这一战略转型的重要佐证。KPMG 是全球四大会计师事务所之一，在 150 多个国家和地区提供审计、税务、咨询服务，此次合作将 Claude 嵌入 KPMG Digital Gateway——这是 KPMG 面向客户的核心数字服务平台。276,000 名员工全员获取 Claude 权限，意味着 KPMG 将 AI 辅助工作从「试点项目」升级为「标准工作方式」，规模远超以往任何企业级 AI 部署案例。

SAP 的合作涉及面更广也更深。SAP 是全球 ERP（企业资源规划）系统的绝对领导者，超过 80% 的世界 500 强企业使用 SAP 管理财务、供应链、人力资源等核心业务。将 Claude 嵌入 SAP Business AI Platform 并作为「主要推理和 Agentic 核心能力」，意味着数以万计使用 SAP 系统的企业客户将在日常工作中接触到 Claude 驱动的 AI 功能——采购预测、财务异常检测、合同分析、HR 流程自动化等场景将以 Claude 为后端引擎。这与 Anthropic 同期发布的 Claude for Small Business（接入 QuickBooks、PayPal、HubSpot 等工具）形成「大中小企业全覆盖」的战略布局。

这两笔合作对澳洲 IT 就业市场有相当直接的影响。KPMG 和 SAP 在澳洲都有大规模业务，KPMG 澳洲员工逾 1 万人，SAP 系统覆盖 ASX 50 中的绝大多数企业。短期内，熟悉 Claude API 和 Anthropic Agent SDK 的开发者在这类企业的 AI 项目中有明显竞争优势；中期来看，「如何在 SAP 或四大咨询环境中落地和管理 AI 解决方案」将成为高价值咨询技能。对于正在准备澳洲 IT 求职的 JR Academy 学员，将「Claude API 实操 + 企业级 AI 集成理解」写进简历和面试准备中，是非常具体的加分项。
