问题

新人用 AI 差，5 个症状

这套训练库是为了治这 5 个症状的。如果你身上没有这些，可以跳过。

1没有身份感

不知道自己角色每天该做什么，更不知道哪件事该让 AI 帮忙。

2不会让 AI 帮 review

稿子写完直接交 manager，AI 只用来生成不用来批评，自己永远学不会判断质量。

3不知道怎么 eval

不知道一段 AI 输出"好不好"的标尺。给 manager 的稿子自己心里没底。

4被 AI 编的"看起来合理"骗

AI 自信地编造"Slack 2018 年 CAC 是 $187"，你信了传给同事，同事一查穿帮。

5不会狂提问

Marketing 沉默就是装懂。新人不问问题 = manager 看不到你在思考什么 = 你在原地。

设计原则

5 条原则 · 看完每个任务你都能理解为什么这么设计

1

不替决策

AI 是放大器，不是替代品。

每个任务的 Step 1 都明写「不准用 AI」。你必须先自己想，AI 才能在你的判断上迭代。

2

必自审

没有自审工具的员工，永远不知道自己做的好不好。

每个产出任务必须跑 1 个现成 skill 自审（/wechat-article-quality 等），75 分以下重写。

3

留演化

Manager 看 prompt 演化，不看最终稿。

提交必须含 prompt v1 → v2 → v3 + 每次为什么改。AI 一次过 = 你没动脑。

4

带验收

没有 30 秒能拍板的 checklist，验收就是走过场。

每个任务结尾验收 ✅ ❌ 不超过 5 条，具体到能数 / 能测，不能写「质量好」这种主观词。

5

绑真实工作

造假场景的训练，员工知道是过家家，不会真投入。

任务跟你本周真要做的事挂钩 — 真活动、真公众号、真 EOI、真 campaign。

红线

4 条红线 · 违反 = 任务不算成

⛔ 这 4 条任何一条没满足，manager 直接打回

  1. 不问问题，任务全部不算成。每个任务必须留 ≥ 3 个真实疑问。Marketing 沉默就是装懂。
  2. 直接交 AI 第一版 = 重做。没有 prompt 演化记录（v1 → v2 → v3）= 不合格。
  3. fact-check 失败的"案例" = 重做。引用任何案例 / 数据 / 公司，必须搜得到来源。AI 编的不算。
  4. AI 味文风 = 重写。出现「在当今快速发展的 / 让我们一起 / 准备好了吗 / 深入探讨 / 全面掌握」 = 整段重写。

D1 · 6 个基础任务

6 维度入门 · 纯 ChatGPT · 完成后能基本不被 AI 骗
D1-01 📚 未知知识获取 ● 已开放 ⏱ 1h

用 AI 学 3 个 Marketing 概念，但不准被它骗

你的身份

Marketing 通用岗。下周开会同事会扔出一堆缩写（CAC / LTV / AARRR / PMF / POSE / RACE / ICP / BoFu / ToFu...），你得跟得上节奏，不能在群里装懂或者尬笑。

今天要做的事

挑 3 个你完全不懂的 marketing 缩写 / 概念，用 ChatGPT 1 小时学到能跟同事讨论的水平。关键约束：每个概念 ChatGPT 必须给一个真实公司案例，并且你要去 Google 验证案例真实存在。

可以从这些里挑（不熟的就是不懂的）：CAC、LTV、AARRR、PMF、POSE 模型、RACE 框架、ICP（理想客户画像）、BoFu / MoFu / ToFu、PLG、CRO、NPS、CSAT、CAC payback period、Activation rate、K-factor、retention cohort。

训练目标

练「逼 AI 给真案例 + 不被 AI 编的『看起来合理』的内容骗」。这是新人最容易翻车的能力 — AI 能流利地编一个不存在的「Airbnb 2018 年 PLG 策略」，你信了就传给同事，同事一查穿帮。

步骤
STEP 1 自己先猜 不准用 AI

挑你的 3 个概念后，每个概念先自己想 2 句话：

  • 这个概念字面听起来在说什么？
  • 你猜它跟你工作哪个环节有关？

写在纸上 / Notion 里。猜错没关系，重点是给自己一个起点。

STEP 2 让 ChatGPT 解释 + 给真案例

把这个 prompt 粘到 ChatGPT（先把 [...] 里换成你挑的 3 个概念）：

我是一家 IT 教育公司的 marketing 新人，正在学习以下 3 个概念：

1. [CAC]
2. [AARRR]
3. [ICP]

对每个概念，请给我：

- 一句话定义（不超过 30 字，不要绕）
- 公式 / 衡量方法（如果有）
- 真实公司案例：必须是真实存在、可在 Google / Wikipedia / 公司官方博客查到的案例。禁止编造。如果你不确定某个案例是否真实，直接说「不确定」，不要瞎写。优先选 Slack / Notion / Airbnb / Duolingo / Canva / Figma 这种留下大量公开 case study 的公司。
- 跟我（IT 教育公司 marketing）的关联：这个概念在我日常工作哪个环节用得上？给一个具体例子。

输出格式：
## 概念 1：[名称]
一句话：...
公式：...
真实案例：[公司] 在 [年份] 通过 [具体动作] 达成 [具体结果]。来源：[公开文章 / 报告 URL]
跟你的关联：...
🚀 在 ChatGPT 跑这步
STEP 3 fact-check（这一步是核心）

针对 ChatGPT 给的每一个真实案例，你必须做一次：

  1. 打开 Google
  2. 搜 [公司名] [关键词] [年份]，例如 Slack PLG 2014
  3. 找到一个真实可访问的来源（Medium / 官博 / Wikipedia / 行业报告）
  4. 把链接贴回你的笔记

翻车场景（必看）：

  • ChatGPT 说「Slack 2018 年 CAC 是 $187」→ Google 搜不到 → AI 编的，不能用
  • ChatGPT 说「Airbnb 用 PLG 增长 300%」→ Airbnb 不是 PLG（marketplace）→ 概念用错
  • ChatGPT 给的链接 404 → 来源失效，重新找

3 个概念里超过 1 个 fact-check 失败：不要用更好听的版本糊弄 manager。回 ChatGPT 让它换案例，把 fact-check 失败的过程也记下来 — 这恰恰是这个任务最值钱的产物。

STEP 4 用自己的话写一遍

关掉 ChatGPT，自己用大白话给同事讲一遍每个概念（写在 Notion 里，每个 100-150 字）。

测试：写完读给自己听，能不能像跟人聊天一样讲清楚？还是 AI 腔（「这个概念至关重要，让我们深入探讨...」）？AI 腔再写一遍。

提交给 manager
  1. prompts.md — 跟 ChatGPT 的完整对话（v1 / v2 / v3）
  2. fact-check.md — 每个概念的 fact-check 过程（搜索关键词 + 找到的 URL + ✅/⚠️/❌）
  3. final.md — 你用自己的话写的 3 个概念解释（每个 100-150 字）
  4. questions.md — ≥ 3 个真实疑问
验收标准（manager 30 秒拍板）
  • ✅ 3 个概念都有可点开的真实来源 URL
  • ✅ 至少 1 个 fact-check 失败的案例被诚实记录
  • ✅ Prompt 至少改过 2 轮，每轮改动可解释
  • ✅ 提问 ≥ 3 个，不是水问题
  • ✅ Final 是「人话」，不是 AI 腔
  • ❌ 案例没贴 URL 或 URL 打不开 → 重做
  • ❌ 提问列表只有 1-2 个 → 重做
  • ❌ 出现 AI 腔 → 重写
  • ❌ 跟 ChatGPT 只聊了 1 轮 → 重做
这一步在教你什么

核心能力：识破 AI 幻觉 + fact-check 习惯。AI 最危险的不是它会犯错，是它自信地犯错 — 流利、合理、引用具体数字，让你以为它知道。新人最容易在这上面翻车。

跑通后你会有 3 个肌肉：(1) 看到 AI 给具体数字 / 案例 → 默认怀疑 → 必查证；(2) 区分「AI 真的知道」 vs 「AI 在硬编」；(3) 把 fact-check 当成工作流的一部分。

你的工作不是用 AI 加速生成内容，是用 AI 加速生成你能验证 + 担保 + 输出的内容。

D1-02 🧠 Marketing sense ○ Phase 2 解锁 ⏱ 45min

看 3 家竞品首页 30 秒，让 ChatGPT 拆解定位

Coursera / MasterClass / Skillshare 各 30 秒，让 ChatGPT 拆「卖什么 / 给谁 / 用什么钩子」，再去验证它说的对不对。练「判断 AI 的归纳是真观察还是脑补」。

D1-03 🪝 标题钩子 ● 已开放 ⏱ 1h

用 AI 写 5 个标题钩子，但你自己得先想 2 个

你的身份

Marketing 通用岗。本周要为一个真活动 / 真公众号文写标题。标题决定 90% 的打开率，正文写得再好，标题没钩到人，零阅读。

今天要做的事

为本周 1 个真实工作场景（活动预热 / 公众号文 / 小红书 / EDM 邮件）想标题。关键约束：你必须先自己写 2 句钩子，再让 ChatGPT 扩到 5 句，最后让 ChatGPT 解释每句靠什么打人，最后你自己做最终选择。

训练目标

练「人在前 AI 在后」+「让 AI 解释判断逻辑」。新人最常犯的错：直接让 AI 出 5 个标题，挑一个看起来顺眼的。这样你永远学不会「什么是好钩子」，下次没 AI 你还是写不出来。

步骤
STEP 1 自己先写 2 句 不准用 AI

打开任何文档，写下：

  • 场景：是什么活动 / 文章 / 产品（30 字）
  • 目标读者：是谁 / 几岁 / 什么处境（"墨尔本 IT 留学生 / 找工作 6 个月没 offer / 焦虑"）
  • 钩子 v1：第一感觉的标题，10-20 字
  • 钩子 v2：换一个角度（v1 是恐惧驱动那 v2 试向往驱动），10-20 字

这一步不准开 ChatGPT。你写得再烂都行，重点是你自己先做第一手判断。

STEP 2 让 ChatGPT 扩到 5 句 + 解释打法 
我在为以下场景写标题：
- 场景：[你的场景描述]
- 目标读者：[读者画像，含年龄 / 处境 / 情绪]
- 渠道：[公众号 / 小红书 / EDM / 落地页]

我自己已经想了 2 句钩子：
1. [你的 v1]
2. [你的 v2]

请你做 3 件事：

(A) 扩到 5 句钩子（保留我的 2 句 + 加 3 句新角度，覆盖至少 3 种打法：数据 / 对比 / 恐惧 / 向往 / 社交认同 / 好奇 / 反常识）

(B) 对每句钩子（含我的）解释：
- 用了哪种打法
- 击中读者哪个具体心理（用读者内心独白说，例：「看到这句他会想：'我也是这样啊...'」）
- 弱点是什么（什么样的读者不会被它打到）

(C) 排序：你认为哪句最可能让目标读者点开？理由？哪句最可能翻车（被误读 / 被反感）？

不要用空话。每个解释必须具体到「读者会怎么想 / 会不会点」。禁止「这个标题很有吸引力 / 能引起读者共鸣」这种废话。
🚀 在 ChatGPT 跑这步
STEP 3 让 ChatGPT 反驳它自己

ChatGPT 给完排序后，追加：

现在反驳你刚才的排序。如果你是一个非常挑剔的资深 marketing 总监，你会怎么挑刺这 5 句钩子？哪句你会直接划掉，理由？

然后告诉我：把你最看不上的那句改一下哪个词 / 哪个结构就能救活？

这一步是练「让 AI 帮你做反向思考」。AI 第一遍永远是对你客气的，逼它扮演挑剔的角色才能挖出真问题。

STEP 4 你自己拍板 + 同事盲测

ChatGPT 给的排序不是答案，是输入。

  1. 看完 ChatGPT 的解释和反驳，你自己心里有第一名 + 第二名
  2. 盲测：把 5 句钩子（不带 ChatGPT 的解释）发给 1-2 个同事 / Buddy，问"你觉得哪句最想点开？为什么？"
  3. 对比同事直觉 vs ChatGPT 推荐 vs 你自己直觉 — 三方一致 = 选它；分歧 = 你做最终判断并写明理由
提交给 manager
  1. prompts.md — 完整对话（含 Step 3 反驳追问）
  2. evolution.md — 钩子演化（v1/v2 + AI 加的 + 反驳后改活的）
  3. blind-test.md — ≥ 1 个同事盲测反馈截图
  4. final.md — 最终选择 + 200 字以内说明为什么
  5. questions.md — ≥ 3 个真实疑问
验收标准（manager 30 秒拍板）
  • ✅ 自己的 v1 + v2 在 ChatGPT 介入之前写好（看 prompts.md 时间戳）
  • ✅ Step 3 反驳追问做了，且改活了至少 1 句
  • ✅ 至少 1 个同事盲测反馈贴了截图
  • ✅ 最终选择理由不是「ChatGPT 推荐」 — 必须有自己的判断
  • ✅ 5 句钩子覆盖至少 3 种打法
  • ❌ Prompt 只有 Step 2 没反驳追问 → 重做
  • ❌ 自己的 v1/v2 是从 ChatGPT 反贴回去 → 直接打回
  • ❌ 没做盲测 → 重做（盲测是核心，不是 optional）
  • ❌ 选最终钩子的理由 < 50 字或纯感觉 → 重写
这一步在教你什么

核心能力：判断「什么是好钩子」的肌肉。新人写不出好标题不是缺创意，是没建立判断标准。让 AI 出 100 个钩子你也挑不出最好的 — 你得先有标尺。

所有约束（自己先写 / 让 AI 解释打法 / 反驳追问 / 同事盲测）都是为了在你脑子里塞一把"好钩子"的标尺：(1) 多种打法的覆盖意识；(2) 读者心理具体化能力；(3) 挑刺习惯；(4) 同事盲测的卫生意识。

跑完这个任务，下次写标题你会自然而然走这套流程，不用想。这才是肌肉。

D3-01 🛠️ Skill engineering ○ Phase 4 解锁 ⏱ 1h

📋 Manager 验收 5 步法（30 秒拍板）

  1. 看 prompt 演化记录（不看最终稿） — 至少改了 2 轮才算合格
  2. 看提问列表 — 没有 ≥ 3 个真实疑问 = 直接打回
  3. 看 fact-check 痕迹 — 引用案例 / 数据要求员工贴搜索证据
  4. 看自审分 — 任务指定的自审 skill ≥ 75 分才算过
  5. 看任务专属 checklist — 每个任务"验收标准"那一栏，5 条以内 30 秒拍板

⚠️ 前置 — manager 自己得会：你自己先把 D1-1 / D1-3 跑一遍。manager 不会用 AI，验收会变成走过场，整个 pilot 翻车。
