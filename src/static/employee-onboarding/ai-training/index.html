问题

新人用 AI 差，5 个症状

这套训练库治这 5 个症状。如果你身上没有这些，可以跳过。

1没有身份感

不知道自己角色每天该做什么，更不知道哪件事该让 AI 帮忙。

2不会让 AI 帮 review

稿子写完直接交 manager，AI 只用来生成不用来批评，自己永远学不会判断质量。

3不知道怎么 eval

不知道 AI 输出"好不好"的标尺。给 manager 的稿子心里没底。

4被 AI 编的"看起来合理"骗

AI 自信地编"Slack 2018 年 CAC 是 $187"，你信了传给同事，同事一查穿帮。

5不会狂提问

Marketing 沉默就是装懂。新人不问 = manager 看不到你在思考什么 = 你在原地。

设计原则

5 条原则 · 每个任务都按这个

1

不替决策

AI 是放大器，不是替代品。

每个任务的 Step 1 都明写「不准用 AI」。你必须先自己想，AI 才能在你的判断上迭代。

2

必自审

没有自审工具的员工，永远不知道自己做的好不好。

每个产出任务必须跑 1 个现成 skill 自审，75 分以下重写。

3

留演化

Manager 看 prompt 演化，不看最终稿。

提交必须含 prompt v1 → v2 → v3 + 每次为什么改。AI 一次过 = 你没动脑。

4

带验收

没有 30 秒能拍板的 checklist，验收就是走过场。

每个任务结尾验收 ✅ ❌ 不超过 5 条，具体到能数 / 能测。

5

绑真实工作

造假场景的训练，员工知道是过家家，不会真投入。

任务跟你本周真要做的事挂钩 — 真活动、真公众号、真 EOI、真 campaign。

红线

4 条红线 · 违反 = 任务不算成

⛔ 这 4 条任何一条没满足，manager 直接打回

  1. 不问问题，任务全部不算成。每个任务必须留 ≥ 3 个真实疑问。Marketing 沉默就是装懂。
  2. 直接交 AI 第一版 = 重做。没有 prompt 演化记录（v1 → v2 → v3）= 不合格。
  3. fact-check 失败的"案例" = 重做。引用任何案例 / 数据 / 公司，必须搜得到来源。AI 编的不算。
  4. AI 味文风 = 重写。出现「在当今快速发展的 / 让我们一起 / 准备好了吗 / 深入探讨 / 全面掌握」 = 整段重写。

D1 · 6 个基础任务

D1-01 📚 未知知识获取 ⏱ 1h

用 AI 学 3 个 Marketing 概念，但不准被它骗

挑 3 个你完全不懂的 marketing 概念（CAC / LTV / AARRR / PMF / ICP / BoFu / PLG / NPS / K-factor 等），用 ChatGPT 1 小时学到能跟同事讨论的水平。关键约束：每个 ChatGPT 给的真实公司案例，你必须 Google 验证存在。

步骤要点
STEP 1 自己先猜 2 句 不准用 AI

每个概念字面听起来在说什么 / 你猜跟你工作哪个环节有关。给自己一个起点。

STEP 2 让 ChatGPT 解释 + 给真案例 
我是一家 IT 教育公司的 marketing 新人，正在学习以下 3 个概念：

1. [概念 1]
2. [概念 2]
3. [概念 3]

对每个概念，请给我：
- 一句话定义（不超过 30 字）
- 公式 / 衡量方法
- 真实公司案例：必须可在 Google / Wikipedia / 公司官博查到。禁止编造，不确定就说不确定。优先选 Slack / Notion / Airbnb / Duolingo / Canva / Figma 这种留下大量公开 case study 的公司。
- 跟我（IT 教育公司 marketing）的关联：日常工作哪个环节用得上？给具体例子。

输出格式：
## 概念 1：[名称]
一句话：...
公式：...
真实案例：[公司] [年份] [动作] → [结果]。来源：[URL]
跟你的关联：...
🚀 打开 ChatGPT
STEP 3 fact-check（核心步骤）

每个 ChatGPT 引用的真实案例 → Google 搜 [公司] [关键词] [年份] → 找到真实可访问的来源。翻车场景：AI 说"Slack 2018 年 CAC 是 $187" / "Airbnb 用 PLG 增长 300%" — 这些大概率是编的。诚实记录 fact-check 失败。

STEP 4 用自己的话写一遍

关掉 ChatGPT，自己用大白话给同事讲一遍每个概念（每个 100-150 字）。读给自己听 — AI 腔（"至关重要 / 让我们深入探讨"）→ 重写。

提交给 manager
  1. prompts.md — 完整对话（v1/v2/v3）
  2. fact-check.md — 每个概念的搜索关键词 + 找到的 URL + ✅/⚠️/❌
  3. final.md — 你用自己的话写的 3 个概念解释
  4. questions.md — ≥ 3 个真实疑问
验收（manager 30 秒）
  • ✅ 3 个概念都有可点开的真实来源 URL
  • ✅ 至少 1 个 fact-check 失败被诚实记录
  • ✅ Prompt 至少改过 2 轮，每轮改动可解释
  • ✅ Final 是「人话」不是 AI 腔
  • ❌ 案例没贴 URL → 重做
  • ❌ 出现 AI 腔 → 重写
这一步在教你什么

核心：识破 AI 幻觉 + fact-check 习惯。AI 最危险的不是它会犯错，是它自信地犯错 — 流利、合理、引用具体数字。你的工作不是用 AI 加速生成内容，是用 AI 加速生成你能验证 + 担保 + 输出的内容。

D1-02 🧠 Marketing sense ⏱ 45min

30 秒看 3 家竞品首页，让 ChatGPT 跟你抢答

每家 30 秒看 Coursera / MasterClass / Skillshare 首页，自己写「卖什么 / 给谁 / 首屏钩子」9 句，再让 ChatGPT 反驳。新人最容易直接让 AI 拆竞品 — 这里要练「自己先归纳，AI 在后面验证」。

步骤要点
STEP 1 每家 30 秒，写 3 句 不准用 AI

计时器，每家关掉前写：卖什么 / 给谁 / 首屏哪句话最抓你。3 家共 9 句。

STEP 2 让 ChatGPT 反驳你的 9 句 
我刚做了一个 30 秒看竞品练习。我看了 Coursera / MasterClass / Skillshare 三家首页。

我的判断：
[贴你的 9 句]

请你不要直接告诉我「正确答案」，而是：
1. 基于训练数据，每家公司真实定位是什么？给具体证据（最近 1 年公开 case study / 新闻 / 财报，必须可 fact-check）
2. 我的 9 句判断里哪几句对，哪几句偏？为什么？
3. 我漏掉了什么这 3 家最重要的差异点？

不要客气，挑刺越狠越好。
🚀 打开 ChatGPT
STEP 3 fact-check ChatGPT 的"证据"

至少 1 条 ChatGPT 引用应该 fact-check 不通过（必然，AI 在这种数据上经常编）。诚实记录"AI 编了 X"。

STEP 4 写 200 字差异对比

"Coursera/MasterClass/Skillshare 三家最大差异 + 匠人哪里跟它们都不一样"。必须有至少 1 处你 30 秒看到但 ChatGPT 没提的细节。

验收
  • ✅ 9 句判断在 ChatGPT 介入前（看时间戳）
  • ✅ 至少 1 个 ChatGPT 引用被你 fact-check 出问题
  • ✅ 差异对比含 1 处你亲眼看到、AI 没提的细节
  • ❌ 9 句是从 ChatGPT 反贴回去 → 重做
  • ❌ 没 fact-check 失败的记录 → 重做

D1-03 🪝 标题钩子 ⏱ 1h

用 AI 写 5 个标题钩子，但你自己得先想 2 个

为本周 1 个真实场景（活动预热 / 公众号文 / 小红书 / EDM）想标题。关键约束：自己先写 2 句钩子，再让 ChatGPT 扩到 5 句，再让 ChatGPT 解释每句靠什么打人，再让 ChatGPT 反驳自己，最后你拍板 + 同事盲测。

步骤要点
STEP 1 自己先写 2 句 不准用 AI

场景 + 目标读者 + v1（第一感觉）+ v2（换一个角度，例如 v1 恐惧驱动那 v2 试向往驱动）。

STEP 2 让 ChatGPT 扩到 5 句 + 解释打法 
我在为以下场景写标题：
- 场景：[你的场景]
- 目标读者：[读者画像，含年龄/处境/情绪]
- 渠道：[公众号/小红书/EDM/落地页]

我自己已经想了 2 句钩子：
1. [你的 v1]
2. [你的 v2]

请你做 3 件事：
(A) 扩到 5 句钩子（保留我的 2 句 + 加 3 句新角度，覆盖至少 3 种打法：数据/对比/恐惧/向往/社交认同/好奇/反常识）

(B) 对每句解释：用了哪种打法 / 击中读者哪个具体心理（用读者内心独白说，例：「看到这句他会想：'我也是这样啊...'」）/ 弱点是什么

(C) 排序：你认为哪句最可能让目标读者点开？理由？哪句最可能翻车？

不要用空话。每个解释必须具体到「读者会怎么想 / 会不会点」。禁止「这个标题很有吸引力」这种废话。
🚀 打开 ChatGPT
STEP 3 让 ChatGPT 反驳它自己 
现在反驳你刚才的排序。如果你是非常挑剔的资深 marketing 总监，你会怎么挑刺这 5 句钩子？哪句你会直接划掉，理由？

然后告诉我：把你最看不上的那句改一下哪个词 / 哪个结构就能救活？
STEP 4 同事盲测 + 你拍板

把 5 句钩子（不带 ChatGPT 解释）发给 1-2 个同事盲测。对比同事直觉 vs ChatGPT 推荐 vs 你直觉，最后你做最终判断。

验收
  • ✅ v1 + v2 在 ChatGPT 介入之前写好（看 prompts.md 时间戳）
  • ✅ Step 3 反驳追问做了，且改活了至少 1 句
  • ✅ 至少 1 个同事盲测反馈贴了截图
  • ✅ 最终选择理由不是「ChatGPT 推荐」
  • ❌ v1/v2 是从 ChatGPT 反贴回去 → 直接打回
  • ❌ 没做盲测 → 重做
这一步在教你什么

判断「什么是好钩子」的肌肉。新人写不出好标题不是缺创意，是没建立判断标准。让 AI 出 100 个钩子你也挑不出最好的 — 你得先有标尺。这套约束（自己先写 / 让 AI 解释打法 / 反驳追问 / 同事盲测）就是建标尺。

D1-04 🎨 海报视觉 ⏱ 45min

给真海报跑 ChatGPT user test，3 个 persona

挑 1 张匠人已发布的活动海报，用 /poster-user-test 让 ChatGPT 扮演 3 个真实匠人 persona，模拟"我看完想不想点 / 报名"。新人评估海报最常说"我觉得挺好的" — 但你不是目标用户。

步骤要点
STEP 1 自己看 30 秒 不准用 AI

3 秒内第一眼看到什么 / 你猜给谁看的 / 会不会点。写完关掉不准回看。

STEP 2 跑 /poster-user-test 三 persona 
/poster-user-test

挑 3 个 persona 跑这张海报：
- 墨尔本 IT 留学生 / 找工作 6 个月 / 焦虑
- 悉尼 Master 在读 / 想转 AI / 时间紧
- 工作 2 年的 Junior dev / 想升职

每个 persona 输出：
1. 第一眼 3 秒看到什么（视觉路径）
2. 心里 OS（"哦这是给我的吗"/"看起来贵不贵"）
3. 行为预测：会停留 / 划过 / 点 / 报名？给百分比预估
4. 这张海报对这个 persona 翻车的点是什么？

不要客气，越真实越好。
🚀 打开 ChatGPT
STEP 3 对比直觉 vs persona 输出 + 写改进 brief

哪 persona 反应让你意外？这反应说明海报哪里没钩到？写 200 字给设计师：保留什么 / 改 1-2 处具体 / 不要动什么。

验收
  • ✅ 30 秒判断在 ChatGPT 介入前
  • ✅ 3 persona 不同质（覆盖至少 2 种处境 / 痛点）
  • ✅ 给设计师反馈是具体的（"第二行字号 +20%"），不是空话
  • ❌ 反馈是"再优化一下" → 重写

D1-05 📈 用户增长 ⏱ 1h

用 AARRR 列匠人 5 个增长 lever，挑 1 个估算 ROI

用 AARRR（Acquisition/Activation/Retention/Referral/Revenue）拆匠人现状，列 5 个可执行 lever，挑 1 个估算成本+影响+失败概率。新人想增长方案最容易飘 — 框架强迫你具体到 5 个不同环节。

步骤要点
STEP 1 自己想匠人最弱的一环 不准用 AI

5 选 1（A 拉新 / A 激活 / R 留存 / R 推荐 / R 收入），写 100 字理由 — 基于这一周看到的什么？

STEP 2 让 ChatGPT 列 5 个 lever 
我是匠人学院（IT 教育公司，澳洲华人为主）的 marketing。我猜我们最弱的一环是 [Step 1 选的环节]。

请按 AARRR 框架，给我 5 个针对这个环节的具体 lever。每个 lever 必须含：
1. 一句话描述（具体到能下周开始执行）
2. 真实案例：哪家公司用过类似 lever？必须可 fact-check
3. 成本估算（人时 + 钱）
4. 预期影响（百分比 + 衡量指标）
5. 失败概率（30%/50%/70%）+ 失败时的 fallback

禁止编造案例。不确定就说"不确定"。
🚀 打开 ChatGPT
STEP 3 fact-check + 挑 1 个写 200 字 proposal

每个 lever 的真实案例 Google 验证。挑最值得试的那个写 200 字（含下周能做什么 / 谁参与 / 衡量指标）。

验收
  • ✅ Step 1 在 ChatGPT 之前完成
  • ✅ 5 个 lever 每个对应真实公司案例（fact-check 通过 ≥ 4 个）
  • ✅ Proposal 含具体下周 action（不是"开始研究"）
  • ❌ Lever 是"做更多投流"这种空话 → 重做

挑本周一个真活动 / 真课程发布，让 ChatGPT 各出恐惧 / 向往 / 社交认同 3 个情绪驱动版本 campaign 大纲，对比哪个最匹配匠人 voice。新人最容易让 AI 直接给 1 个方案 — marketing 决策本质是 N 版本之间选最优。

让 ChatGPT 扮演挑剔总监挑刺：哪个最像 AI 写的 / 哪个最容易翻车 / 哪个最匹配匠人。你拍板 + 100 字理由（含落地细节）。

验收
  • ✅ Step 1 直觉在 ChatGPT 介入前
  • ✅ 3 版本 hook 句明显不同
  • ✅ 反驳追问做了，且改活了至少 1 处
  • ❌ 3 版本互相像 → 重做

D2 · 10 个真实输出任务

6 维度真实产出 + 反思 · D2-10 反思最重要 ● 全部已开放
D2-01 🎯 Campaign 策划 ⏱ 2h

写下周真活动的完整 Campaign 方案

为下周真活动写完整 7 部分 campaign 方案：目标 / 受众 / 核心信息 / 3 渠道分发 / CTA / KPI / 失败 fallback。禁止造假场景。自审用 /webinar-topic-feasibility（线上）或 /offline-event-content-design（线下）。

步骤要点
STEP 1 自己写 ICP + 核心信息 不准用 AI

ICP（不是"年轻人"，是"墨尔本 IT 留学生 / 已申请 PR / 没工作经验"）+ 核心信息 1 句话。

STEP 2 让 ChatGPT 扩成完整方案 
我要为以下活动做 campaign 方案：
- 活动：[名字]
- 时间地点：[具体]
- ICP：[Step 1 写的]
- 核心信息：[Step 1 写的]

请扩成完整 campaign 方案，按这 7 部分：
1. 目标（数字化的，例：30 人报名 + 15 人到场 + 5 个 EOI）
2. 受众（主受众 / 次要受众 / 排除谁）
3. 核心信息（3 个不同表达版本）
4. 3 渠道分发（公众号 / 小红书 / LinkedIn 任选 — 标题 + 时间 + 配图风格）
5. CTA（不同渠道用不同 CTA — 不要全用"扫码报名"）
6. KPI（每个渠道独立可衡量，不是"提升品牌"）
7. 失败 fallback：报名不到 5 人 / 到场 < 50% 怎么办

每部分必须具体到「下周谁干什么」。禁止"提升品牌"这种废话。

把 D3-01 找出的不匹配规则真改了 — 改完跑改前 / 改后对比，提 PR 让团队 review。能挑出问题不算本事，能改好让团队接受才是。

步骤要点
STEP 1 备份原版 + 改 1-3 行

在 Claude Code 里：拷贝原文件备份 → 改 D3-01 找出的那条 → git diff 确认只改了那 1 处。不顺手"再改一下别的"。

STEP 2 用同一篇文章跑改前 / 改后对比

预期：总分变化 < 5（说明你只改了一个点）+ 目标维度分数变化跟预期方向一致 + 没意外伤害其他维度。

STEP 3 写 PR description + 提 PR

结构：为什么改（具体翻车场景）/ 改了什么（diff）/ 改前后对比（数字）/ 风险讨论。让 Claude Code 帮你创建 branch + commit + push + 给 PR URL。

验收
  • ✅ Git diff 只有 1-3 行改动
  • ✅ 改前 / 改后对比有数字
  • ✅ PR description 含「为啥改 + 风险讨论」
  • ✅ PR 真的提了（有 URL）
  • ❌ Diff > 5 行 → 重做（改动太大）
  • ❌ 没提 PR 只是本地改 → 不算完成

D3-03 🛠️ Skill engineering ⏱ 2h

写 1 个 mini skill，覆盖现有没覆盖的场景

挑 1 个匠人现有 skill 没覆盖的真实场景（群公告 / LinkedIn 帖 / 朋友圈文案 / 招聘 JD / EDM / 用户回复），按模板写最小可用版 SKILL.md，跑 3 篇真稿件验证，提 PR。这是最稀缺的能力 — 大多数人只会用工具，会写工具的人极少。

步骤要点
STEP 1 选场景 + 自己写 5 维度 先不动 ChatGPT

每维度：名字（1 个名词）/ 一句话定义 / 满分。5 个维度互相独立 + 覆盖核心痛点。

STEP 2 让 ChatGPT 反驳 + 改维度

挑刺：哪些维度重叠 / 哪些太主观员工评分会差很大 / 漏了什么核心维度 / 5 个是不是太多太少。至少改 2 处。

STEP 3 写 SKILL.md（参考已有 skill 结构）

含：什么时候用 / 不用 / 5 维度（每个有评分锚点 + 红线 + 改进建议）/ 输出格式表格。

STEP 4 跑 3 篇真稿件验证 + 提 PR

3 篇分数有区分度（差 ≥ 20 分）+ 改进建议具体（不是"再优化"）。提 PR 含设计依据 + 风险讨论。

验收
  • ✅ 维度 5 个（不超过 9 个）
  • ✅ 至少改了 2 处（v1 vs v2 不同）
  • ✅ 跑 3 篇真稿件有区分度
  • ✅ SKILL.md 含「输出格式」段
  • ✅ PR description 含「设计依据 + 风险」
  • ❌ 维度互相重叠 → 重做
  • ❌ 跑 3 篇分数全一样 → 区分度不够，重做

跑完 D3-03，你已从「ChatGPT 用户」走到「skill 工程师」 — 这是 marketing 新人能在 1 个月内达到的最高水平之一。

Manager 验收

收到提交后按这 5 步看

📋 Manager 验收 5 步法（30 秒拍板）

  1. 看 prompt 演化记录（不看最终稿） — 至少改了 2 轮才算合格
  2. 看提问列表 — 没有 ≥ 3 个真实疑问 = 直接打回
  3. 看 fact-check 痕迹 — 引用案例 / 数据要求员工贴搜索证据
  4. 看自审分 — 任务指定的自审 skill ≥ 75 分才算过
  5. 看任务专属 checklist — 每个任务"验收"那一栏，5 条以内 30 秒拍板

⚠️ 前置 — manager 自己得会：你自己先把 D1-1 / D1-3 跑一遍。manager 不会用 AI，验收会变成走过场，整个 pilot 翻车。
