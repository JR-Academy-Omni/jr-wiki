## 1. SoftBank 750亿欧押注法国，5GW AI 数据中心布局欧洲

![SoftBank AI data center infrastructure](https://images.unsplash.com/photo-1558494949-ef010cbdcc31?w=800)

**一句话**: SoftBank 宣布在法国建设史上最大欧洲 AI 基础设施，最高 750 亿欧元、5GW 算力，2031 年前首批 3.1GW 上线。

5 月 31 日，SoftBank Group 正式宣布将在法国投入最高 750 亿欧元（约合 875 亿美元）建设 AI 数据中心，目标到 2031 年在法国北部 Hauts-de-France 大区实现 5 吉瓦（GW）算力容量落地。一期计划在 Dunkirk（Loon-Plage）、Bosquel 和 Bouchain 三地交付 3.1GW，合作伙伴为法国工业巨头 Schneider Electric，后者将在 Dunkirk 帮助建立大规模工业生产集群。孙正义在发布会上将这笔投资定性为「SoftBank 对人工智能革命的核心押注」，法国总统马克龙亲身出席见证签署。

这项投资的体量几乎超过此前欧洲所有 AI 数据中心建设投资的总和。对比来看：微软 2024 年在欧洲的 AI 基础设施承诺约 40 亿欧元，谷歌在德国的云数据中心投资约 30 亿欧元，而 SoftBank 单次出手就是 750 亿欧元，规模差距超过 15 倍。背后的逻辑是：随着推理模型（Reasoning Model）的普及，单次对话所消耗的算力是传统 Transformer 模型的数十倍，训练 + 推理的叠加需求将在未来三到五年内推动全球算力需求出现另一个量级的跃升。SoftBank 在法国占据地理优势——Hauts-de-France 大区拥有丰富的可再生能源接入条件和跨大西洋海底电缆节点。

对于正在学习 IT 和 AI 方向的 JR Academy 学员而言，这一投资信号明确：欧洲 AI 相关的云基础设施、DevOps、数据中心运维岗位将在未来两到三年内出现大规模增量需求。法国本土大学（巴黎综合理工、格勒诺布尔大学等）的 AI 方向毕业生将迎来前所未有的就业市场，而在澳洲有欧盟居留权的 IT 从业者或可考虑欧洲就业选项。此外，SoftBank 在 OpenAI 的重要投资人身份使二者数据中心资源共用成为可能，未来基于法国节点的低延迟欧洲 AI API 服务值得长期关注。

## 2. OpenAI Codex 上线 Windows 视窗操控，Goal Mode 正式 GA

![OpenAI Codex computer use interface](https://images.unsplash.com/photo-1607252650355-f7fd0460ccdb?w=800)

**一句话**: Codex 新增 Computer Use for Windows，能看屏幕、点击、打字操控 PC 应用，并可从 iOS/Android 远程延续工作流；Goal Mode 同步正式 GA。

OpenAI 在最新 Codex 更新中推出了 Computer Use for Windows 功能：Codex 现在可以实时「看到」Windows 屏幕截图，并基于视觉理解在 Windows 应用中执行点击、输入、拖拽等操作，完成此前需要人工手动操作的界面交互任务。更引人注目的是跨设备延续能力——用户在 PC 上启动一个 Codex 工作流后，即使离开电脑，也可以通过 iOS 或 Android 上的 ChatGPT App 继续监控和推进任务。与此同时，Goal Mode 宣布正式进入 General Availability（GA）阶段，覆盖 Codex App、IDE Extension 和 CLI 三大入口，允许用户定义最终目标和成功标准，让 Codex 在无需逐步指令的情况下持续自主工作。

Computer Use for Windows 背后的技术意义在于：它将 Codex 从「纯代码生成工具」升级为「真正的桌面 Agent」。此前，Codex 即使生成了完美的代码，最后的「执行 + 验证」步骤仍需人工介入——比如在 IDE 里跑测试、在浏览器里验证结果、在 Windows 服务中重启进程。Computer Use 将这些步骤纳入 Codex 的能力范围，意味着一个完整的软件开发迭代循环（写代码 → 执行 → 测试 → 看结果 → 修 bug）可以在 Codex 内闭环完成。与 Anthropic Claude 的 Computer Use 功能相比，OpenAI 在时间上稍晚，但 Windows 场景的针对性更强，对企业 IT 环境（80% 以上仍运行 Windows）的适配更直接。

对开发者的实际影响分两类：一是短期效率提升——CI/CD 流水线中那些需要人工点几下界面才能触发的步骤，现在可以完全自动化；二是长期角色转变——重复性操作工作（如 QA 手工测试、数据录入验证、IT 运维脚本执行）将被 Codex 这类 Agent 大量替代，IT 从业者需要将精力转向定义目标、设计工作流、评审 Agent 输出质量。JR Academy 的 IT 课程中关于自动化测试和 DevOps 流水线的内容，正是在这个背景下变得愈发核心。

## 3. Meta 测试 $7.99 付费 AI 订阅，Conversations 大会 6 月 3 日揭晓更多

![Meta AI subscription WhatsApp Messenger](https://images.unsplash.com/photo-1611162617474-5b21e879e113?w=800)

**一句话**: Meta 在新加坡等地测试 Meta One Plus（$7.99/月）和 Meta One Premium（$19.99/月），6 月 3 日 Conversations 大会将发布 WhatsApp/Messenger AI 新功能。

Meta 在 5 月 27 日确认，公司已开始在新加坡、危地马拉和玻利维亚三个市场小范围测试付费 AI 订阅服务：Meta One Plus 定价 7.99 美元/月，Meta One Premium 定价 19.99 美元/月。这是 Meta 历史上首次在消费者端推出直接的 AI 付费订阅产品，是对「非广告收入」的重要探路。就在这一测试启动的数天后，Meta 年度消息大会 Conversations 2026 将于 6 月 3 日举行，届时将集中展示 WhatsApp、Messenger 和 Instagram Direct 的 AI 功能升级路线图——付费版与免费版的功能差异大概率届时公开。

Meta 选择在新加坡首批测试绝非偶然：新加坡是东南亚 AI 渗透率最高的市场之一，当地用户对高品质数字服务的付费意愿显著高于周边市场，且法规环境相对宽松，有利于快速迭代产品策略。对比来看，$7.99 的起步价低于 ChatGPT Plus（$20）和 Claude Pro（$20），更接近大众消费定价区间——这与 Meta 旗下 WhatsApp 和 Messenger 数十亿月活用户的基本盘相匹配：哪怕转化率只有 1%，也意味着数以千万计的付费用户。

Meta 的付费订阅战略对整个 AI 行业的影响不可低估。首先，它证明 AI 助手的「直接付费」模式在移动端具备可行性，会加速其他社交/消息平台效仿；其次，Meta One 若与 WhatsApp Business 打通，将给中小企业提供极具吸引力的 AI 客服和营销工具，直接威胁 Intercom、HubSpot 等 SaaS 工具；第三，Meta AI 背后的 Muse Spark 模型（由 Meta Superintelligence Labs 研发）的商业化窗口将因此提前开放。对于 JR Academy 学员，在新加坡就读或求职的同学可在第一时间体验并评测，这是一份有价值的求职加分项目——真实使用后的产品对比报告在面试中极为有说服力。

## 4. xAI Grok Build 0.1 编程模型 API 公测开放

![xAI Grok developer API coding](https://images.unsplash.com/photo-1461749280684-dccba630e2f6?w=800)

**一句话**: xAI 将旗下最快编程模型 Grok Build 0.1 通过 xAI API 向开发者开放公测，无需 SuperGrok 或 X Premium+ 订阅。

xAI 于 5 月 29 日宣布，Grok Build 0.1 正式进入 xAI API 公开测试阶段。这款定位为「xAI 最快编程模型」的产品此前仅限于 SuperGrok 订阅用户或 X Premium+ 订阅者通过 Grok Build CLI 使用，公测开放意味着任何拥有 xAI API 账户的开发者都可以直接通过 API 调用构建自己的编程辅助工具或将其集成进开发流水线。此次开放与 Grok V9 Medium（1.5 万亿参数，预计 6 月中旬发布）的训练完成公告几乎同步，显示 xAI 在「旗舰推理模型」和「轻量编程专用模型」两条产品线上同步发力。

从竞争格局来看，编程 AI 工具市场已是强敌林立：GitHub Copilot 坐拥 GitHub 生态优势，Cursor 以 IDE 集成深度闻名，Claude Code 以推理质量见长，Gemini Code Assist 借助 Google Cloud 整合。Grok Build 0.1 在这个战场上的差异化定位是「速度」——xAI 强调其响应延迟和代码输出速度优于同类模型，这对于需要高频代码补全（而非复杂推理）的开发者场景有实际价值。xAI 还同步推出了 Connectors 功能，可将 SharePoint、Outlook、OneDrive、Google Workspace、Notion、GitHub 和 Linear 等工具直接集成进 Grok 工作流，这让 Grok Build 不只是一个「补全工具」，而是可以读取项目上下文再生成代码的「上下文感知 Agent」。

对于正在学习 Full Stack 或 DevOps 课程的 JR Academy 学员，Grok Build 0.1 API 公测提供了一个零成本（或极低成本）的测试机会：将其与日常使用的 Copilot 或 Cursor 进行代码补全质量、响应速度、错误修复准确率三个维度的横向对比，并整理成 1-2 页的测评报告，是一份在求职简历上极有说服力的实战经历。Grok V9 Medium 发布后（约 6 月中旬），Build 模型很可能基于新旗舰底座升级，届时能力有望大幅提升，持续关注 API 更新日志是跟上节奏的最佳方式。

## 5. Google DeepMind 联手美国能源部，发起 Genesis AI 科学国家使命

![Google DeepMind scientific research AI](https://images.unsplash.com/photo-1507413245164-6160d8298b31?w=800)

**一句话**: 白宫发起 Genesis 国家计划，以 AI 加速联邦科学研究；Google DeepMind 作为关键合作伙伴支持美国国家实验室，布鲁克海文国家实验室为首批接入方。

美国能源部正式宣布 Genesis 计划——一项旨在将 AI 能力系统性融入联邦政府科学研究体系的国家使命，涵盖物理学、化学、材料科学、生命科学等多个领域。Google DeepMind 受邀成为关键合作伙伴，提供其领先的 AI 模型和计算资源，支持包括布鲁克海文国家实验室（Brookhaven National Laboratory）在内的联邦研究机构开展前沿研究项目。DeepMind CEO Demis Hassabis 同期披露，Google 在 6 月份处理的 Token 量接近一千万亿（1 quadrillion），相比 5 月环比翻倍以上，印证了 AI 推理需求的指数级增长正在真实发生。

Genesis 计划的历史意义可以从多个维度理解。在科研层面，此前 AI 对科学的最大贡献是 AlphaFold 系列对蛋白质结构预测的革命，而 Genesis 是将这种「点状突破」系统化为「国家科研基础设施」——意味着 AI 辅助将从「个别实验室的探索」变成「联邦政府日常科研流程的标配组件」。在政治层面，这是拜登之后美国政府继续推进 AI 战略优先权的重要信号，与此同时 Google DeepMind 通过与政府深度绑定进一步巩固了其在「科学 AI」领域的领导地位，与 OpenAI 更偏向消费级和企业级的商业化路径形成明显分化。

对于有志于科技行业的 JR Academy 学员，Genesis 计划释放的就业信号值得长期跟踪。一方面，美国联邦政府和国家实验室系统将大幅增加 AI 相关的研究工程师、数据科学家、模型部署与评估专家等岗位；另一方面，Genesis 的成功实施将推动澳洲、英国等盟友国家跟进类似项目——澳洲的 CSIRO 和多所 Group of Eight 高校与能源部合作机构有密切联系，相关方向的研究项目可能随之增加。此外，DeepMind 在此次合作中所涉及的「科学计算 + AI 结合」技术栈（大型模拟、分子动力学、量子计算辅助），是目前就业市场上稀缺性最高、薪资溢价最大的技能组合之一。
